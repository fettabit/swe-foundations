# Building Large Software Projects

# Optimize for:
- dependability 
- extendability 
- team scaleability 
- development velocity 

# Reduce risk
- Determine the risk profile 
	- basically the things that are most likely to fail. 
- Think about:
	- Changing platforms
	- Language implementations
	- Changing hardware 
	- Changing priorities 
	- Loosing customers 

> Many small chances of failure == a big change of failure 

### Key takeaways
- write unbreakable code…even if you have to be explicit. 
- Write code so well you don’t have to go back to it. 
- modularize software 

#### Notes 
- Modules are black boxes (should be)
- one module = one person 
- One person = possibly many modules
- Learn to think forward 
- Avoid implementing good enough for now
